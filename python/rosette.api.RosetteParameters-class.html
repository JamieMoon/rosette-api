Class RosetteParameters

                      RosetteParameters

Parameter object for all operations requiring input other than translated_name. Four fields, content, contentType, unit, and inputUri, are set via the subscript operator, e.g., params["content"], or the convenience instance methods RosetteParameters.load_document_file and RosetteParameters.load_document_string. The unit size and data format are defaulted to InputUnit.DOC and DataFormat.SIMPLE.

Using subscripts instead of instance variables facilitates diagnosis.

If the field contentUri is set to the URL of a web page (only protocols http, https, ftp, ftps are accepted), the server will fetch the content from that web page. In this case, neither content nor contentType may be set.

load_document_file(self, path, data_type='application/octet-stream')

source code 

Loads a file into the object. The file will be read as bytes; the appropriate conversion will be determined by the server. The document unit size remains by default InputUnit.DOC.

Parameters:
  • path - Pathname of a file acceptable to the open function.
  • data_type (DataFormat) - One of DataFormat.HTML, DataFormat.XHTML, or DataFormat.UNSPECIFIED. No other types are acceptable at this time, although HTML is broad enough to include text strings without markup. If the data type is unknown, or describes a binary file, use the default (DataFormat.UNSPECIFIED).

load_document_string(self, s, data_type)

source code 

Loads a string into the object. The string will be taken as bytes or as Unicode dependent upon its native python type and the data type asked for; if the type is HTML or XHTML, bytes, not python Unicode, are expected, the encoding to be determined by the server. The document unit size remains (by default) InputUnit.DOC.

Parameters:
  • s - A string, possibly a unicode-string, to be loaded for subsequent analysis, as per the data_type.
  • data_type (DataFormat) - The data type of the string, as per DataFormat.
